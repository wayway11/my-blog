最近 Claude Code 的源码在网络上被泄露，作为一个一直在用 Claude Code 写代码的人，看完之后有几点想法。

## 不只是工具，是一套工作流

泄露的代码里最让我意外的不是模型调用逻辑本身，而是围绕它构建的那一整套工程体系 — 任务编排、上下文管理、工具调用链、错误恢复机制。这些代码揭示了一个事实：**大模型应用的核心竞争力早就不在模型层了，而是在于怎么把模型能力工程化**。

Claude Code 里面那套 sub-agent 调度系统就是一个很好的例子。把一个大任务拆成小步骤，每个步骤交给一个独立的 agent 去执行，执行完再汇总 — 这种设计比"一个超大 prompt 一把梭"稳定得多。

## 工程细节决定体验

源码里能看到很多对体验影响巨大的设计决策：

- **上下文窗口的利用策略** — 不是无脑塞满，而是做分层摘要和选择性遗忘，保证关键信息不丢失
- **工具调用的幂等性** — 每个操作都要考虑到重试场景，文件编辑用精确匹配而不是行号
- **沙箱隔离** — bash 命令的执行是隔离的，不会污染宿主环境

这些东西用户看不见，但缺了任何一个，体验就会断崖式下降。

## 这对我们有什么启发

第一，**不要只关注模型本身的进展**。GPT-5、Claude 5 这些当然重要，但真正决定一个 AI 产品好不好用的，是它怎么把模型能力组织起来的"脚手架"。

第二，**好工具是被打磨出来的**。泄露的代码里能看到大量的边界情况处理，这些都是踩过坑之后才加上的。没有任何一个 AI 工具的第一版是完美的。

第三，**用 AI 写代码的时候，要理解它在做什么**。Claude Code 本身的代码质量很高，但仍然有一些 hacky 的地方。AI 生成的代码需要人来把关，这一点不会变。

## 写在最后

源码泄露当然是不好的，但从来技术圈的历史来看，每一次重大泄露都会带来一波学习浪潮。与其纠结泄露本身，不如好好研究这些代码的设计思想，用到自己的工作里去。

这大概就是"站在巨人的肩膀上"在 AI 时代的新含义吧。
